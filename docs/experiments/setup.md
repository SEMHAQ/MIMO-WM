# 实验设置

## 两个数据集，两个目的

### 合成数据集（脚本自生成）

用 `scripts/generate_expanded_dataset.py` 脚本生成，不是公开数据集。状态转移规则为当前状态的线性变换加上动作耦合和一个很小的非线性项（tanh，系数0.1），每条episode使用不同的随机参数。

属于**近线性动力学**——非线性项系数仅0.1，线性部分占主导。

### MuJoCo Humanoid（公开仿真基准）

使用MuJoCo物理仿真器的Humanoid-v4环境，包含真实的接触动力学、摩擦力和碰撞响应，是人形机器人领域最接近真实的公开基准。

### 为什么要两个？

| | 合成数据集 | MuJoCo Humanoid |
|---|---|---|
| **目的** | 验证计算效率和MPC集成 | 验证预测精度 |
| **动力学** | 近线性（简单） | 真实接触+摩擦+碰撞（复杂） |
| **状态维度** | 376 | 376 |
| **动作维度** | 17 | 17 |
| **数据量** | 500条episode（400训练/100验证） | 200条episode |

## 对比方法

- **LSTM-WM**：经典基线，2层和4层版本
- **Transformer-WM**：注意力机制，精度好但慢
- **Mamba-WM**：SSM的热门变体，需要CUDA算子
- **线性回归**：最简单的baseline，验证非线性建模的必要性

## 评价指标

- **MSE**：预测误差，越小越好（单位 ×10⁻³）
- **R²**：模型解释了多少方差，越接近1越好
- **推理时间**：越快越好（ms）
- **参数量**：越少越好（M = 百万）
