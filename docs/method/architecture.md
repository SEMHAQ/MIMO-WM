# SSM-WM 架构

## 整体思路

把世界模型想象成一个"三明治"：

```
输入：历史状态+动作序列
    ↓
┌─────────────────────┐
│  编码器：压缩到隐空间  │  ← 把376维状态+17维动作压缩到128维
├─────────────────────┤
│  SSM主干：建模时序关系  │  ← 4层SSM块，用FFT高效处理序列
├─────────────────────┤
│  解码器：还原状态      │  ← 从128维隐空间还原回376维状态
└─────────────────────┘
    ↓
输出：预测的下一时刻状态
```

## 核心创新：SSM块的结构

每个SSM块做四件事：

```
输入 → ①归一化 → ②SSM递推 → ③门控过滤 → ④残差连接 → 输出
```

- **①归一化（LayerNorm）**：对每个时间步的特征做标准化，让数值稳定在合理范围，训练更容易收敛
- **②SSM递推**：核心计算。用FFT（快速傅里叶变换）把整个序列一次性并行处理，而不是一步一步算。这就是SSM比LSTM快的关键
- **③门控（sigmoid）过滤**：用sigmoid函数（输出0到1之间的值）生成门控信号，类似LSTM的遗忘门，选择性保留有用信息。这是我们加的关键组件
- **④残差连接**：把输入直接加到输出上，防止深层网络训练时梯度消失。没有它，4层SSM块很难训练

堆叠4层这样的块，模型就能捕捉从短期到长期的时序依赖。

## 门控机制：为什么重要？

没有门控 → SSM对所有信息一视同仁
有门控   → SSM能选择"这个信息重要，保留"或"这个噪声，丢弃"

消融实验证明：去掉门控后精度下降2-5%，在复杂接触动力学场景下降更大。

## 训练方式：单步 + 多步

- **单步损失**：直接比较预测值和真实值
- **多步损失**：用模型自己的预测结果继续往后推（展开8步），和真实轨迹比较

多步损失让模型学会"自己的误差会怎么累积"，对MPC的多步展开预测至关重要。

## 计算复杂度

训练时用FFT卷积 → O(T log T)
部署时用递推模式 → O(1) 单步延迟

两种模式数学等价，只是实现方式不同。
